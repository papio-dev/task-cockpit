# MassiveParallelDeps-workspace

Стресс-тест для массового параллельного запуска зависимостей.

## Сценарий

Главная задача **"Run All Workers"** запускает **100 воркеров параллельно** через `dependsOn` + `dependsOrder: "parallel"`.

Каждый воркер выполняется **3-10 секунд** (случайное время).

## Что тестируется

- Корректное отслеживание состояния 100+ одновременных задач
- Производительность UI при массовых обновлениях статусов
- Утечки памяти при большом количестве терминалов/процессов
- Race conditions в event handlers
- Корректное завершение главной задачи после всех зависимостей

## Ожидаемое поведение

1. При запуске "Run All Workers" — мгновенно стартуют все 100 воркеров
2. В tree view — 100 элементов переходят в состояние "running"
3. Воркеры завершаются в случайном порядке (кто раньше — кто позже)
4. После завершения последнего воркера — выполняется главная команда
5. Общее время ~10 сек (параллельно), не ~500 сек (последовательно)

## Метрики для проверки

- CPU/Memory usage во время выполнения
- Задержка обновления UI (должна быть < 100ms)
- Отсутствие "зависших" статусов после завершения
